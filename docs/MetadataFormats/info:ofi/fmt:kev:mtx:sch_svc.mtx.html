A representation of a Key/Encoded-Value pair is generated by concatenating the contents of the first four columns of a row that begins with an ampersand in the Matrix below. The ordering of KEV pairs is not important. Rows that have '#' in the first column are comments and should not be included in the representation.

The following data types are provided for the values of the Keys, which must be URL-encoded:

<data> Character string
<id> Character string for an Identifier, of the form: [ ori:([a-zA-Z0-9]:)+:[^:]+ | uri:.+ | xri:.+ ]
<fmt-id> Character string for a Format Identifier, of the form: [ ori:fmt:[a-z]:[a-z]:[^:]+ | uri:.+ | xri:.+ ]
<url> Character string for a URL
<m-key> Character string for a Metadata Key, of the form: [a-zA-Z0-9]+
<date> Character string representing a date according to W3CDTF profile of ISO 8601, of the form: [ YYYY-MM-DD | YYYY-MM | YYYY ]

Abbreviations in column headings:

  • Delim - Delimiter
  • Min - minimum occurrence
  • Max - maximum occurrence ('*' = unbounded)

In the Value column:

  • "yes" means that the service is requested
  • "no" means that the service is not requested
Delim Key Equals Value Min Max Description
& abstract = "yes", "no" 0 1 This ServiceType refers to an abstract as it is commonly referred to in the scholarly information community, e.g. a brief resume of the content of a text or document. Its intended purpose is to request an abstract associated with the Referent of the ContextObject, e.g. the abstract for a journal article.
& citation = "yes", "no" 0 1 This ServiceType refers to a citation as it is commonly referred to in the scholarly information community, e.g., the bibliographic citation of an article or book. Its intended purpose is to request a citation associated with the Referent of the ContextObject, e.g. the citation for a journal article.
& fulltext = "yes", "no" 0 1 This ServiceType refers to full text as it is commonly referred to in the scholarly information community. Its intended purpose is to request the text associated with the Referent of the ContextObject. For example, if the Referent were a journal article, the request would be for the full text of the journal article. If the Referent is a book, the request could be for an e-book or an excerpt of the book.
& holdings = "yes", "no" 0 1 This ServiceType refers to holdings information as it is commonly referred to in the scholarly information community. Here, Holdings refers to the materials owned by a library. In the case of serially published items it also means the range of published issues currently possessed by the library. The standard does not specify the level of detail at which holdings are reported. The intended purpose of this ServiceType is to request holdings information associated with the Referent Entity, e.g. to find libraries that hold copies of the Referent.
& ill = "yes", "no" 0 1 This ServiceType refers to Interlibrary Loan as it is commonly referred to in the scholarly information community. Its intended purpose is to request an Interlibrary Loan associated with the Referent Entity in the ContextObject.
& any = "yes", "no" 0 1 This ServiceType refers to any ServiceType.
